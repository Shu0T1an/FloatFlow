# FloatFlow 产品需求文档（PRD）

**版本：** V1.1  
**状态：** 设计确认中  
**更新时间：** 2026-04-07

## 1. 产品定位

FloatFlow 是一个轻量、低打扰、可随时呼出的桌边效率挂件，核心能力聚焦在两件事：

- `待办`：快速记录和查看当前任务
- `便笺`：快速记录灵感、备注和临时信息

它不是一个“大而全”的桌面工作台，而是一个可以固定在桌面边角、在需要时一键呼出、在工作过程中始终陪伴用户的微型效率工具。

## 2. 当前设计方向

### 2.1 产品形态

- 首版优先做成 `桌边挂件` 形态，而不是大面板应用
- 默认固定在屏幕 `右下角`
- 支持通过快捷键呼出 / 隐藏
- 支持 `Pin` 后常驻显示
- 后续再扩展自由拖动、贴边停靠、半收起等能力

### 2.2 命名统一

为贴近用户心智，界面文案统一使用：

- `待办` 对应 Todo
- `便笺` 对应 Memo

开发实现内部仍可保留 `todo / memo` 等技术命名，但产品层统一以中文呈现。

## 3. 目标用户与核心场景

### 3.1 目标用户

- 开发者：随手记接口、任务、调试点、灵感
- 设计师：快速记录交互想法、文案、视觉调整项
- 内容与运营：收集碎片信息、待处理事项、临时备注
- 学生与轻办公用户：在学习或浏览资料时边看边记

### 3.2 核心使用场景

- 正在工作时，突然想到一条任务，按快捷键呼出窗口后立刻记录
- 需要持续提醒时，把 FloatFlow `Pin` 在桌面边角常驻显示
- 工作中临时保存一段想法或备注，写入 `便笺`
- 在桌面侧边快速查看当前进行中的任务，而不切换到其他重型应用

## 4. 核心交互定义

### 4.1 窗口行为

- 默认是一个小型桌边挂件窗口
- 首次启动默认显示在屏幕 `右下角`
- 支持 `Alt + Space` 呼出 / 隐藏主窗口
- 支持 `Pin` 开关
  - 开启时：窗口置顶并保持常驻
  - 关闭时：窗口仍可被快捷键呼出，但不强制持续显示

### 4.2 首版窗口特征

- 单窗口
- 小尺寸、紧凑布局
- 不追求复杂多栏布局
- 优先保证“看一眼就懂、按一下就记”

### 4.3 桌面模式优先级

首版优先级如下：

1. 右下角桌边挂件
2. 快捷键呼出 / 隐藏
3. Pin 常驻置顶
4. 透明度调节
5. 后续再扩展可拖动、自定义位置、贴边停靠

## 5. 信息架构

首版界面由 5 个层级组成：

1. 顶部栏  
品牌、Pin、呼出状态、关闭/隐藏入口

2. 模式切换区  
`待办 / 便笺` 双标签切换

3. 桌面控制区  
透明度滑条、轻量搜索/筛选入口

4. 主内容区  
根据当前模式展示待办列表或便笺内容

5. 底部状态栏  
本地同步状态、数据导出、系统设置等轻量入口

## 6. 待办页设计

### 6.1 目标

待办页是 FloatFlow 的高频主界面之一，强调：

- 快速新增
- 快速勾选完成
- 对“进行中”和“已完成”有清晰分组
- 视觉上更像桌边任务挂件，而不是完整任务管理软件

### 6.2 结构

- 顶部 `待办 / 便笺` 切换
- 透明度控制条前置
- 主输入条始终靠前
- 列表分为：
  - `进行中`
  - `已完成`
- 已完成项目弱化显示，但不立即消失
- 保留“清理已完成”能力

### 6.3 交互

- 在输入框中按 `Enter` 直接创建任务
- 点击圆形勾选区切换完成状态
- 支持基础优先级表达
- 高优先级以更强视觉提示显示，但整体风格仍然克制

## 7. 便笺页设计

### 7.1 设计目标

便笺页不是传统的大编辑器，而是更轻量的桌边灵感区。  
重点是“快速记下瞬时灵感”，而不是复杂排版。

### 7.2 当前确认的视觉方向

基于参考图，便笺页采用：

- 与待办页一致的紧凑挂件外壳
- 顶部 `待办 / 便笺` 切换
- 透明度滑条前置
- 中部放置一个强提示的“记录瞬时灵感”入口
- 下方使用 `便笺卡片列表` 展示已有内容

### 7.3 便笺内容结构

每张便笺卡片至少包含：

- 标题
- 简短正文预览
- 更新时间

首版建议默认行为：

- 点击“记录瞬时灵感”或便笺卡片进入编辑
- 编辑内容自动保存
- 列表页优先展示最近更新的便笺

### 7.4 首版交互原则

- 便笺以“卡片列表 + 当前编辑入口”为主
- 不做复杂富文本
- Markdown 预览能力可以保留，但不作为首页主视觉重点
- 用户第一眼应感知为“轻量便笺工具”，而不是“文档编辑器”

## 8. 视觉与风格规范

### 8.1 总体方向

- 保持当前 FloatFlow 的配色气质，不整体切换为另一套品牌风格
- 布局和密度参考轻量桌边挂件
- 强调低打扰、可长期常驻、桌面融合感

### 8.2 界面特征

- 小尺寸卡片式主窗
- 轻毛玻璃 / 柔和发光边框
- 紧凑圆角
- 稍高信息密度
- 控件尺寸偏小，但仍保证可点击性
- 动效克制，避免厚重感

### 8.3 风格要求

- 不做传统后台面板感
- 不做重度文档编辑器感
- 不做“过于网页化”的松散布局
- 要更像桌面挂件、边角工具、常驻助手

## 9. MVP 功能范围

### 9.1 必做

- 单窗口桌边挂件
- 默认右下角显示
- `待办 / 便笺` 双模式
- `Alt + Space` 呼出 / 隐藏
- `Pin` 常驻置顶
- 透明度调节
- 待办快速新增、完成、清理
- 便笺快速新增、编辑、自动保存
- 本地 JSON 持久化
- 数据导出

### 9.2 暂不实现

- Side Dock 自动贴边收起
- Widget Mode 深度桌面层嵌入
- 图片拖拽
- 全局搜索
- 富文本编辑器
- SQLite
- 多窗口
- 自定义自由拖动记忆位置

## 10. 非功能要求

- 启动后视觉上应足够轻，不产生重应用感
- 响应要快，呼出后可立即输入
- 本地持久化稳定，异常 JSON 不应导致崩溃
- 桌面操作不应打断用户当前主任务

## 11. MVP 验收标准

- 应用启动后以桌边挂件形态显示
- 窗口默认位于右下角
- `Alt + Space` 可稳定呼出 / 隐藏
- `Pin` 可控制常驻置顶
- 待办页可快速新增并分组展示任务
- 便笺页可展示便笺卡片并进入编辑
- 透明度调整即时生效
- 关闭后重开，待办、便笺、窗口偏好仍保留

## 12. 后续演进方向

- 自由拖动与位置记忆
- 边缘停靠与自动收起
- 快速捕获独立小窗
- 更完整的便笺组织方式
- 搜索与标签过滤
- 更深的桌面集成能力
