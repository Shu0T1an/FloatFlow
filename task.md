# FloatFlow Task Board

## 1. 背景与目标

当前 FloatFlow 已从“大面板工具”重构为右下角桌边挂件。
本阶段目标不是继续扩功能，而是把现有挂件整理成一个可验收、可继续迭代的稳定版本。

本轮稳定化重点：
- 修正透明度语义
- 增强 `Pin` 与右下角吸附反馈
- 补齐设置浮层信息
- 清理用户可见中文乱码
- 收口验证链路

## 2. 当前范围

本阶段已纳入：
- 右下角桌边挂件布局
- `Todo / 便笺` 双模式
- `Pin`
- 挂件视觉透明度
- `Alt + Space` 呼出/隐藏
- `Alt + N` 快速进入主录入入口
- JSON 持久化
- 设置浮层基础说明

本阶段明确不做：
- 拖动
- 位置记忆
- 自由吸附
- 便笺删除
- 便笺归档
- 更复杂的桌面模式

## 4. Issue / 问题板

### 用户 Issue
- [x] 透明度到 `100%` 时，为什么还会出现透明效果，不应该是实色吗？
- [x] 没找到右下角吸附按钮，当前是不是默认吸附？现在是不是还不支持拖动？
- [x] `Pin` 按钮之前缺少明显反馈，是否已经做成可感知的双态？
- [ ] 双屏幕是否存在问题，当我在桌面按下跨借鉴的时候无法正常的显示，请进行排查

### 当前处理结论
- [x] 透明度语义已修正为“挂件内容层视觉透明度”，`100%` 对应实色观感
- [x] 当前版本没有单独的“吸附按钮”，而是默认显示时自动吸附右下角
- [x] 当前版本仍不支持拖动，这一点已在设置浮层中明确说明
- [x] `Pin` 已改为明确双态控件，开启时高亮，关闭时中性态

### 待继续确认
- [ ] 在真实 Tauri 运行环境中确认 `100%` 时的实际视觉效果是否完全符合预期
- [ ] 在真实桌面环境中确认用户是否能明确理解“默认吸附右下角”的行为

## 5. 风险与限制

- 当前还没有完成真实桌面环境下的手动验收，现阶段结论主要来自测试、构建和 `cargo check`
- 当前版本位置固定，不支持拖动、位置记忆或自由吸附
- 设置浮层仍是轻量版本，只覆盖本阶段必要信息
- 若后续加入拖动，当前“默认吸附右下角”的表达与交互逻辑需要重新设计
